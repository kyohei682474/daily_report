## わかったこと 
* varは関数スコープなのでスコープ外でも扱うことができる
  * 関数スコープ(関数内)であればどこでも使用することができる
* letは{}であれば使用することができる
* Recordとはキーと値の型を指定したオブジェクト型を作るユーティリティ
* Record単体で見ればオブジェクト型であるし、その中身は<>ないで指定することができる
  ```javascript
  type A = Record<string, number>;
  type A = {[key: string]: number;};
  ```
* extendsは〜の制約でなければダメというという制約
  * T extends Record<string, unknown>はTはオブジェクト型でないとダメですということ
  * type TypeOf<T extends Record<string, unknown>このようにすることでなんでもTypeOfに入れて型定義することを防ぐことができる
* keyofはキーを取り出してユニオン型にして返す演算子
* Mapped Typesはキーをループして新しい型を作成する
  * 基本構文としては{ [K in キーのユニオン]: 値の型 }
  * キーを1つずつ取り出して新しいオブジェクト型を作成するための仕組み
* Excludeはユニオンから除外したい型を除外する。
* ```Exclude<ユニオン型, 除外したい型>```
* neverはありえないという型を示している
* ```type Example = {onClick?: never;};```はonClickがあるならばnever型という意味
* 実質的はこのプロパティは存在してはいけないという形になってしまう
